# 闪电式产品需求文档 (Lightning PRD)

## 1. 产品概述

### 1.1 产品愿景与使命
**愿景**: 让每个人都能成为数据科学家，通过简单的交互完成复杂的数据分析与建模任务。
**使命**: 降低数据科学门槛，提供从数据清洗到因果推断的一站式智能化解决方案，帮助企业和个人快速挖掘数据价值。

### 1.2 价值主张
- **极速洞察**: 自动化流程替代繁琐的手工编码，分析效率提升 10 倍。
- **透明可信**: 不仅给出预测结果，更提供深度的因果解释，解决 AI "黑盒" 信任问题。
- **全流程覆盖**: 打通数据接入、治理、建模、评估到解释的完整链路。

### 1.3 目标平台
- **Web 端**: 现代浏览器 (Chrome, Edge, Safari)，响应式设计适配桌面及平板。

### 1.4 核心假设验证结果 (基于 MVP)
- **痛点确认**: 用户对自动化数据清洗和模型解释性有强烈需求。
- **解决方案匹配**: "LimiX" 的可视化流程能有效降低操作难度。

## 2. 用户研究

### 2.1 用户画像
- **核心用户**: 业务分析师 (Business Analyst)
  - **特征**: 懂业务逻辑，不懂复杂代码，依赖 Excel/BI 工具，渴望预测能力。
  - **痛点**: 现有 AutoML 工具太复杂，缺乏结果解释。
- **次级用户**: 初级数据科学家
  - **特征**: 需要快速构建 Baseline，验证想法。
  - **痛点**: 重复造轮子，数据清洗耗时。

### 2.2 核心使用场景
1.  **销售预测**: 导入历史销售数据，预测未来销量，并分析影响销量的关键因子（如价格、促销）。
2.  **客户流失分析**: 识别高风险流失客户，并通过因果推断找到挽留策略。
3.  **数据质量体检**: 快速扫描数据集，发现缺失、漂移等问题并自动修复。

### 2.3 痛点与需求层次
- **Basic (必须有)**: 准确的模型预测，支持常见数据格式 (CSV/Excel)。
- **Performance (越多越好)**: 自动化清洗程度，模型训练速度。
- **Delight (超出预期)**: 交互式因果图谱，"What-if" 反事实推理模拟。

## 3. 功能需求 (MVP & V1.0)

### 3.1 功能清单与优先级

| 模块 | 功能点 | 优先级 | 描述 |
| :--- | :--- | :--- | :--- |
| **数据资产** | 数据导入 | P0 | 支持 CSV/Excel 拖拽上传，自动解析 Schema |
| **数据资产** | 版本管理 | P1 | 数据集的多版本快照管理 |
| **数据评估** | 质量体检 | P0 | 缺失值、分布漂移、异常值检测与可视化报告 |
| **数据清洗** | 自动清洗 | P0 | 基于规则的自动清洗建议与一键执行 |
| **推理任务** | 任务创建 | P0 | 选择目标列、时间列，配置 AutoML 参数 |
| **推理任务** | 状态追踪 | P0 | 实时展示训练进度、耗时与状态 |
| **推理结果** | 性能评估 | P0 | 回归/分类指标展示 (AUC, RMSE, F1 等) |
| **推理结果** | 模型对比 | P1 | 与竞品 (AutoGluon, LLM) 的性能对比表格 |
| **因果解释** | 全局解释 | P0 | 特征重要性排序 (Shapley Value) |
| **因果解释** | 归因分析 | P1 | 单样本预测归因，展示各特征对结果的贡献度 |

### 3.2 核心用户故事
1.  作为业务分析师，我希望上传 Excel 表格后能立即看到数据质量报告，以便我知道数据是否可用。
2.  作为运营经理，我希望能一键训练预测模型，并看到影响结果的前 5 个关键因素，以便制定运营策略。
3.  作为初级算法工程师，我希望看到不同模型的性能对比，以便选择最优基线。

## 4. 非功能需求

### 4.1 性能要求
- **响应速度**: 页面加载 < 2s，交互响应 < 100ms。
- **处理能力**: MVP 阶段支持 100MB 以内 CSV 文件流畅处理。

### 4.2 安全需求
- **数据隐私**: 用户数据仅在本地/会话期间处理（演示版），或加密存储。

### 4.3 可用性标准
- **零代码**: 全程无须编写 Python/SQL 代码。
- **引导式交互**: 复杂流程提供 Step-by-Step 指引。

## 5. 技术考量

### 5.1 技术栈建议
- **前端**: React + Vite + Tailwind CSS (快速开发，生态丰富)。
- **UI 组件**: Arco Design (企业级交互) + Recharts (数据可视化)。
- **Mock 方案**: 前端拦截请求或本地 Mock 数据，确保演示稳定性。

### 5.2 约束条件
- **开发周期**: 4 周完成 MVP (已完成基础版)。
- **资源限制**: 暂无后端支持，需通过高保真 Mock 模拟真实体验。

### 5.3 核心数据模型
- `Dataset`: { id, name, version, schema, health_score }
- `Task`: { id, type, status, target_col, metrics, created_at }
- `Model`: { id, name, type, params, performance }
